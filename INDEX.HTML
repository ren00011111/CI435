Week 1: ...

This week's workshop involved the completion of the setup process of a simple webpage. This webpage will be used to document my learning journey throughout the course, and will be updated weekly with new content. Today, a clearly structured webpage was established within a space on Brighton Domains, with a header, navigation menu, main content area, and footer. The webpage content was structured using HTML5 semantic elements, such as header, image and anchor elements. This created a logicaL hierarchy of content, which is important for accessibility and search engine optimisation.

Further Learning

In additon to this week's university workshop, I have been exploring the use of semantic HTML elements in web design. This includes understanding how to use HTML elements to create a logical structure for webpages. I have also been researching best practices for accessibility and SEO, which are important considerations when building webpages. A resource that I found particularly helpful was the freeCodeCamp Responsive Web Design Certification – Cat Photo App Workshop. This was due to the immediate implementation of each concept that was covered, which allowed me to apply my learning in a practical way. The workshop provided clear instructions and examples, which made it easier to understand how to use semantic HTML elements effectively in web design, as opposed to passively reading documentation. This active learning approach also encouraged experimentation, error recognition and correction, which overall enhanced my understanding of the material.

After completing the ascending levels of freeCodeCamp's workshop, I was presented the task of building a basic webpage from scratch. This was a great opportunity to apply the knowledge I had gained from the workshop, and to further solidify my understanding of semantic HTML elements. The direct translation of theoretical understanding into practical application was a key factor in my learning process, due to being a visual learner: it allowed me to see the actual implications of the concepts I had learned. This process involved trial and error, through the use of appropriate heading levels, the importance of correctly nesting elements, and the overall architecture of the webpage.

Being aware of the specific learning techniques that work best for me has been instrumental in my learning journey. As a visual learner, I find that I learn best through seeing and doing, rather than through passive reading. This has allowed me to tailor my approach to learning web design, as I have sought out resources and opportunities that allow me to apply my learning in a practical way. This lead me to install the Live Server extension in Visual Studio Code, which allows me to see the immediate results of my code changes in a web browser. This has been a valuable tool for my learning, as it allows me to quickly identify and correct errors, and to see the impact of my changes in real time. This also reduced the gap between coding and output, directly strengthening my confience as I progressed.

Week 2:

This week's workshop involved the adaptation of elements previously established within a simple webpage. The webpage's adaptations included the addition of lists, tables, and definition lists, as well as the implementation of images and links. These elements were added to enhance the structure and content of the webpage, and to provide a more comprehensive learning experience. The addition of lists allowed for the organisation of content in a clear and concise manner, while the implementation of tables provided a way to display data in a structured format. The use of definition lists allowed for the presentation of terms and their definitions in a clear and organised way. The inclusion of images and links added visual interest to the webpage, and provided opportunities for further exploration of related content. Overall, these adaptations enhanced the learning experience by providing a more comprehensive and engaging webpage, while also allowing for the application of new concepts and techniques in web design. We were also introduced to the concept of CSS, which is used to define the presentation of HTML elements. This was a new concept for me, and I found it interesting to learn about how CSS can be used to control the layout and appearance of a webpage. Due to this, the aesthetics of my webpage are continously growing, which is a great motivator for me to continue learning and improving my web design skills. This is achieved through the use of CSS properties such as color, font, and layout, held in an external stylesheet, allowing for the creation of visually appealing and user-friendly webpages, as well as manageable code.

Week 5: DRAFT

This week's workshop allowed me to develop a clearer understanding surrounding the importance of testing and evaluation within the web development process. The lecture introduced the standards-compliant processes regarding the implementation of forms. Despite this task appearing simple, I quickly learnt it is actually something that requires strenious planning and consideration. This approach allows us to bridge the gap between the developer's perspective and the needs of the end user when interacting with the website. We were also equipped with tools such as W3C Markup Validation and CSS Validation Services, alongside built-in browser development tools to debug HTML and CSS, allowing us to finalise a strong foundation in both languages and prepare for the introduction of Javascript. A practical issue I encountered involved missing images when uploading files to the server. Today, I learnt that whilst images may appear correctly when referencing a local file path, this changes once the site is uploaded online.
